## Issues Available for Contributors

I have created 8 labelled issues ready for classmates and contributors to pick up.
They are all sitting in the **Backlog** on the GitHub Project board — unassigned and open.

### How to claim an issue:
1. Browse the open issues: https://github.com/Mabotse-Mosima/UniMatch-University-Application-System/issues
2. Pick one that interests you — start with anything labelled `good-first-issue` if you are new to the codebase
3. **Leave a comment on the issue saying you are working on it** — this stops two people doing the same thing
4. Fork the repo, create a branch, make your changes, and open a Pull Request
5. Reference the issue number in your PR description (e.g. `Closes #66`)

> Issues are **not assigned to anyone** by default. First person to comment and submit a PR gets the credit!
